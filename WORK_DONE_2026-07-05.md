## 1. 완료된 작업 목록

### ✅ 작업 A: 유량 cascade 재계산 결함 수정 (핵심)
**파일**: `server/routes/flowRoutes.cjs` (recalculateFlowTypeCascade)
**현장 증거**: 7/5 로그 `POST /api/flows/bulk -> 200 | calculated_flow:-294720` (마이너스 그대로 저장)
**원인**:
- 기존: 이미 `calculated_flow`가 있으면(마이너스 포함) raw를 바꿔도 그대로 보존 → "수정이 안 되고 있어"
- `explicitDates`가 "사용자가 건드린 날짜" 전부를 보존 대상으로 분류

**수정 내용**:
1. **raw 기반 자동 계산 분기 (87-95번 줄)**: `isExplicit` 조건 제거. raw가 바뀌면 `prevRaw` 기준 재계산. 단 `is_manual`(수동 유량), `is_reset`(초기화), `imported`/`baseline`(엑셀 임포트/관리자 지정)은 보존 유지.
2. **슬러지 누적 분기 (75-86번 줄)**: 같은 결함. 반출량 변경 시 `prevSludgeCumulative` 기준 재계산하도록 `isExplicit && hasStoredFlow` 분기 제거.
3. **마이너스 클램프 (사용자 결정 반영)**: `raw < prevRaw`이면 `flow=0`. 사용자가 "경고·허용 안 한다, 전날 검침값을 수정하겠다"고 확정.
4. **`isExplicit` 변수 제거**: 더 이상 사용 안 함.

**기대 효과** (사용자 시나리오):
- 7/3 raw 비어있고 7/4 raw=1000 → 7/4 calculated=1000-이전raw
- 7/3 raw=900 채워 넣으면 → 7/3 calculated=900-7/2raw, **7/4 calculated=1000-900=100 자동 재계산**
- 마이너스 발생 시 0 처리, 전날 raw 수정하면 자동 양수 복원

### ✅ 작업 C: 진단 로그 app_version 버그 수정
**파일**: `electron/main.cjs` + `server/services/diagnosticLogService.cjs`
**현장 증거**: 7/2~7/5 모든 진단 로그의 `app_version`이 `?` (1.0.9/1.0.10 구분 불가)
**원인**: `getAppVersion()`이 `require('../../package.json')`을 쓰는데, asar 패키징 시 `package.json`은 asar 안에만 있고 서버 cwd(`app.asar.unpacked`)엔 없음 → require 실패 → 빈 문자열.

**수정 (옵션 A: 환경변수 주입)**:
- `electron/main.cjs`: 서버 fork 시 `env.OSOO_APP_VERSION = app.getVersion()` 추가
- `diagnosticLogService.getAppVersion()`: `process.env.OSOO_APP_VERSION` 우선 확인 후 require fallback

**기대 효과**: 1.0.11 이후 진단 로그에 정확한 버전 기록 → 원격 진단 가능.

### ✅ 작업 D: single instance lock 추가
**파일**: `electron/main.cjs`
**현장 증거**: 7/5 트레이에 앱 아이콘 2개 뜸 (사용자가 한 개 수동 종료)
**원인**: `requestSingleInstanceLock()` 코드가 아예 없었음 → 두 프로세스가 같은 DB/포트 잡고 충돌 위험.

**수정**: `app.whenReady()` 이전에 단일 인스턴스 락 추가. 두 번째 실행은 종료, 기존 창 포커스.

> `electron/` 수정은 `AGENTS.md`상 사용자 확인 항목. 본 수정은 Electron 표준 패턴(구조 변경 아님)이나, 최종 승인은 사용자가 담당 에이전트와 확인 예정.

### ✅ 작업 E: 자동업데이트 정책 변경 (사용자 핵심 요구)
**파일**: `electron/updater.cjs` + `electron/main.cjs` + `electron/preload.cjs` + `src/App.jsx`
**사용자 요구**: "로그인 안 된 상태(트레이)에서도 자동 설치 + 사용자 로그인 시 '업데이트를 실행합니다' 안내 후 설치"
**배경**: 양방향 현장 1명 관리, 오전 9시 동시 로그인 불가 → 로그인 전 자동 설치 필수.

**수정 1단계 — idle 자동 설치 (updater.cjs)**:
- `powerMonitor.getSystemIdleTime()` 단일 기준으로 5분 idle 감지 (1.0.9의 `before-input-event` 의존 결함 개선)
- `update-downloaded` 이벤트에서 `scheduleIdleInstall()` 시작
- 30초 폴링, idle 5분 도달 시 `installDownloadedUpdateAndQuit()` (이때 `stopServerGracefully()`로 안전 종료 — main.cjs에 이미 구현됨)
- 환경변수 `OSOO_UPDATE_INSTALL_IDLE_MS`(기본 5분), `OSOO_UPDATE_INSTALL_POLL_MS`(기본 30초)로 조정 가능

**수정 2단계 — 로그인 시 pending 안내 모달 (App.jsx)**:
- `app:getUpdateStatus` IPC 추가 (main.cjs) + `getUpdateStatus` preload 노출
- App.jsx: 로그인 완료(`isAuthenticated`) 시 `getUpdateStatus()` 호출
- 다운로드 완료된 업데이트가 있으면 모달: "새 버전 업데이트가 다운로드되었습니다. 지금 설치하고 앱을 다시 시작하시겠습니까?"
- "지금 설치" 클릭 시 `installUpdate()`, "나중에" 클릭 시 모달 닫기 (idle 자동설치는 계속 대기)

**기대 흐름**:
- 밤새 트레이 → idle 5분 도달 시 자동 설치 (1단계)
- 아침에 사용자가 먼저 앱을 만지면 자동 설치 미뤄지더라도, 로그인 직후 안내 모달로 설치 유도 (2단계)
