# 공사 입력 도우미 기능 계획

## 1. 배경

현장관리자는 앱에 유량, 약품, 키트 데이터를 입력하더라도 도로공사 홈페이지에 별도로 접속하여 같은 성격의 값을 다시 입력하고 일지를 업로드해야 한다.

기존에는 엑셀 파일을 보면서 도로공사 홈페이지에 옮겨 적었지만, 앱 중심으로 업무를 바꾸면 오히려 현장관리자가 앱과 홈페이지를 번갈아 확인해야 해서 더 불편해질 수 있다.

따라서 일지 메뉴 하위에 `공사 입력 도우미` 메뉴를 추가하고, 현장관리자가 도로공사 홈페이지를 브라우저에서 클릭한 상태에서도 앱의 도우미 창을 계속 볼 수 있게 하여 값을 쉽게 옮겨 적도록 한다.

## 2. 목표

- 일지 메뉴 하위에 `공사 입력 도우미` 메뉴를 추가한다.
- 메뉴 클릭 시 별도 모달 또는 항상 위에 표시되는 보조 창을 띄운다.
- 도우미 창은 도로공사 일일운영일지 화면처럼 유량 현황, 전력량 현황, 약품 사용현황을 한 화면에 섹션으로 표시한다.
- DatePicker로 기준일을 선택하면 로컬 DB의 해당 데이터를 바인딩한다.
- 현장관리자가 도로공사 홈페이지 입력 순서대로 값을 확인하고 옮겨 적을 수 있게 한다.
- 향후 도로공사 홈페이지 입력 양식 순서가 확인되면, 같은 순서의 그리드를 제공하여 한 번에 복사/붙여넣기할 수 있게 확장한다.

## 3. 사용자 흐름

```text
1. 현장관리자 로그인
2. 일지 메뉴 진입
3. 하위 메뉴에서 공사 입력 도우미 클릭
4. 도우미 창 표시
5. 기준일 선택
6. 유량, 전력량, 약품 사용현황 섹션 확인
7. 앱이 로컬 DB 데이터를 조회해 표시
8. 현장관리자는 브라우저의 도로공사 홈페이지를 보며 값을 옮겨 적음
9. 필요 시 그리드 내용을 복사하여 도로공사 홈페이지에 붙여넣음
```

## 4. 1차 기능 범위

1차 버전은 자동입력까지 가지 않고, 수동 입력 보조에 집중한다.

- 메뉴 추가: `일지 > 공사 입력 도우미`
- 항상 위에 보이는 도우미 창 또는 모달
- 기준일 DatePicker
- 유량 현황 섹션
- 전력량 현황 섹션
- 약품 사용현황 섹션
- 로컬 DB 데이터 바인딩
- 복사용 그리드 영역
- 빈 데이터는 `-` 또는 빈칸으로 표시

## 5. 섹션별 표시 데이터

### 유량 현황 섹션

표시 항목:

```text
전일검침
금일검침
처리량(사용량)
월간누계
연간누계
```

데이터 출처:

- 로컬 DB 유량 데이터
- 필요 시 BigQuery 복구 로직으로 누락 데이터 보완

계산 기준:

- 전일검침: 선택일 전날 검침값
- 금일검침: 선택일 검침값
- 금일유량: 금일검침 - 전일검침 또는 기존 저장 유량값
- 월합산: 선택일이 속한 월의 누적 유량
- 년합산: 선택일이 속한 연도의 누적 유량

### 전력량 현황 섹션

표시 항목:

```text
전일지침
금일지침
사용량
```

데이터 출처:

- 로컬 DB 유량 데이터 중 `전력` 계열 항목

### 약품 사용현황 섹션

표시 항목:

```text
구입량
사용량
월간누계
연간누계
잔량
비고
```

데이터 출처:

- 로컬 DB 약품 입고/사용/재고 데이터
- 약품관리대장 데이터
- 약품입고일지 데이터

키트 사용량은 도로공사 화면의 약품 사용현황 표 안에 함께 표시한다.

## 7. 항상 위에 보이는 창 처리

목표는 사용자가 브라우저의 도로공사 홈페이지를 클릭하더라도 도우미 창이 가려지지 않도록 하는 것이다.

검토 옵션:

### 옵션 A: Electron 별도 BrowserWindow

- `alwaysOnTop: true`
- 작은 보조 창으로 표시
- 브라우저와 앱 사이를 오가도 도우미가 위에 유지됨
- 가장 목적에 잘 맞음

주의:

- `electron/main.cjs`, `electron/preload.cjs` 수정이 필요할 수 있으므로 AGENTS 지침에 따라 실제 구현 전 사용자 확인이 필요하다.

### 옵션 B: 앱 내부 모달

- 구현이 쉽다.
- 앱 내부에서는 잘 보이지만 외부 브라우저를 클릭하면 앱 전체가 뒤로 갈 수 있다.
- 도로공사 홈페이지를 보면서 옮겨 적는 목적에는 부족할 수 있다.

결론:

- 1차 설계는 옵션 A를 목표로 한다.
- 단, 릴리즈 직전 리스크를 줄여야 한다면 옵션 B로 먼저 만들고 다음 버전에서 Electron always-on-top 창으로 확장한다.

## 8. 복사/붙여넣기 확장

도로공사 홈페이지의 실제 입력 순서가 확인되면 그 순서대로 그리드 컬럼과 행을 맞춘다.

확장 방향:

- 탭별 복사 버튼 제공
- 선택한 셀 범위 복사
- 전체 행 복사
- 도로공사 홈페이지 입력 순서와 동일한 TSV 형식 복사
- 여러 칸에 한 번에 붙여넣을 수 있도록 탭 문자와 줄바꿈 형식 사용

예:

```text
항목    구매    사용    재고    월사용량    연사용량
PAC     20      3       17      12          85
```

## 10. 구현 순서

1. 도로공사 홈페이지 입력 순서와 필요한 항목을 현장 화면 기준으로 정리한다.
2. `roadwork-helper` feature 폴더를 만든다.
3. 일지 메뉴 하위에 `공사 입력 도우미` 메뉴 슬롯을 추가한다.
4. 앱 내부 모달 또는 보조 창 UI를 만든다.
5. 기준일 DatePicker와 탭 UI를 만든다.
6. 유량 데이터 바인딩 API를 만든다.
7. 약품 데이터 바인딩 API를 만든다.
8. 키트 데이터 바인딩 API를 만든다.
9. 복사용 그리드와 복사 버튼을 만든다.
10. 도로공사 홈페이지 실제 입력 순서 확인 후 그리드 순서를 조정한다.
11. 항상 위에 보이는 Electron 보조 창으로 확장한다.

## 11. 리팩토링과의 관계

이 기능은 설정 메뉴 리팩토링 전에 계획과 메뉴 공간을 잡아두는 것이 좋다.

이유:

- 일지 메뉴 하위 구조에 새 도우미 메뉴가 추가된다.
- 유량/약품/키트 데이터 바인딩 공통 로직이 필요하다.
- 향후 설정 메뉴에서 도로공사 입력 양식 순서/매핑을 관리할 수 있다.
- 나중에 자동입력 기능으로 확장할 때 현재 구조가 막히지 않아야 한다.

다만 최초 릴리즈 전에는 자동입력까지 구현하지 않는다.

## 12. 완료 기준

- 일지 메뉴 하위에 `공사 입력 도우미` 진입점이 확보된다.
- 기준일 선택 후 유량/약품/키트 데이터가 표시된다.
- 현장관리자가 브라우저를 보면서 값을 옮겨 적을 수 있다.
- 복사용 그리드 구조가 준비된다.
- 도로공사 홈페이지 입력 순서가 확인되면 그리드 순서를 쉽게 바꿀 수 있다.
- `npm run validate`와 `npm run build`가 통과한다.
