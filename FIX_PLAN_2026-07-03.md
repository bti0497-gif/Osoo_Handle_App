### 🔴 장애 1: HWPX/PDF 내보내기 500 — `ESCAPE '\'` 버그

**상태**: 확정 · 1분 수정 가능

**원인**: `server/services/dailyWorkLogHwpxService.cjs:251-252` 가 **백틱 템플릿 리터럴(`` ` ``)** 안에 `ESCAPE '\'` 를 사용.

```js
// ❌ 현재 (dailyWorkLogHwpxService.cjs:251-252) — 백틱 리터럴
const activeFlowNames = db.prepare(`
  SELECT item_name FROM config_items
  WHERE category = 'flow' AND is_active = 1
    AND item_name NOT LIKE '%\_raw' ESCAPE '\'   // ← 백틱 안 → SQL엔 ESCAPE '' (빈문자열) → SQLite 거부
    AND item_name NOT LIKE '%\_flow' ESCAPE '\'
  ORDER BY display_order, id
`).all();
```

**비교 — 다른 모든 파일은 일반 문자열 + `ESCAPE '\\'` 패턴으로 정상 동작**:
- `server/routes/roadworkHelperRoutes.cjs:65-67`
- `server/services/settings/defaultSettingsService.cjs:5`
- `server/services/settings/appSettingsService.cjs:70`
- `server/services/settings/mappingSettingsService.cjs:93`
- `server/database.cjs:681`

일반 문자열 `"ESCAPE '\\'"` 는 JS에서 `\\` → `\` 로 처리되어 SQL엔 `ESCAPE '\'` 로 전달됨 (정상).
백틱 리터럴 `` `ESCAPE '\'` `` 는 이스케이프 미처리 → SQL엔 `ESCAPE ''` 로 전달됨 (에러).

**해결 방안** (2가지 중 택일):

**옵션 A (권장 — 다른 파일과 일관성)**: 일반 문자열로 변경
```js
const activeFlowNames = db.prepare(
  "SELECT item_name FROM config_items WHERE category = 'flow' AND is_active = 1 " +
  "AND item_name NOT LIKE '%\\_raw' ESCAPE '\\' " +
  "AND item_name NOT LIKE '%\\_flow' ESCAPE '\\' " +
  "ORDER BY display_order, id"
).all();
```

**옵션 B (백틱 유지)**: 백틱 안에서 이스케이프 명시
```js
const activeFlowNames = db.prepare(`
  SELECT item_name FROM config_items
  WHERE category = 'flow' AND is_active = 1
    AND item_name NOT LIKE '%\\_raw' ESCAPE '\\'
    AND item_name NOT LIKE '%\\_flow' ESCAPE '\\'
  ORDER BY display_order, id
`).all();
```

**영향도**: 일지작성 → HWPX/PDF 내보내기 즉시复活. 현장에서 가장 많이 겪은 장애.

**검증**: `npm run validate`, 그리고 빌드 후 HWPX/PDF 내보내기 1회씩 수동 테스트.

### 🔴 장애 2: 유량 매핑 500 — 시작행 날짜 검증 실패 시 전체 롤백

**상태**: 확정 · 구조적 개선 필요

**원인**: `server/services/settings/mappingSettingsService.cjs:58-71` 의 `collectMappedDates()`.

```js
// mappingSettingsService.cjs:58-71
function collectMappedDates(rows, startRow, endRow, dateCol, label) {
  const dates = [];
  const seen = new Set();
  for (let r = startRow; r <= endRow; r += 1) {
    const formatted = formatRowDate(getRangeCell(rows, r, dateCol));
    if (!formatted || seen.has(formatted)) continue;
    dates.push(formatted);
    seen.add(formatted);
  }
  if (dates.length === 0) {
    throw new Error(`${label} 날짜를 확인할 수 없습니다. 날짜 칼럼(${dateCol})과 시작행/종료행을 다시 확인해주세요.`);
  }
  return dates;
}
```

현장 로그: 사용자가 시작행 369로 설정 → A369 셀에 날짜 없음 → `dates.length === 0` → throw → **전체 매핑 임포트 롤백**.

문제점:
1. 시작행 1개만 검증 실패해도 31일분 전체가 날아감.
2. 매핑 설정(config_items) 저장은 트랜잭션 밖이라 **설정은 저장됐지만 데이터는 안 들어오는 불일치 상태** 발생.
3. 에러 메시지가 친절하지만, "어느 행부터 날짜가 있는지"를 알려주지 않음.

**해결 방안**:

(a) **자동 시작행 탐색 제안**: `collectMappedDates` 실패 시, 시작행 ±N 행을 스캔해 첫 유효 날짜 행을 찾고 에러 메시지에 제안.
```js
function findFirstDateRow(rows, startRow, endRow, dateCol, scanRange = 30) {
  for (let r = Math.max(2, startRow - scanRange); r <= Math.min(endRow + scanRange, startRow + scanRange); r++) {
    if (formatRowDate(getRangeCell(rows, r, dateCol))) return r;
  }
  return null;
}
// collectMappedDates 에서 dates.length === 0 일 때:
//   const suggestion = findFirstDateRow(...);
//   throw new Error(`시작행(${startRow})에 날짜가 없습니다. ${suggestion ? `가까운 날짜 행: ${suggestion}` : '날짜 칼럼/시트를 확인하세요.'}`);
```

(b) **매핑 설정 저장과 임포트 분리** (문제 5와 연계): 날짜 검증 실패 시 **설정(config_items, app_settings)은 이미 저장된 상태로 두고**, 임포트만 실패 처리. 사용자가 시작행만 고쳐서 재시도하면 됨.

**영향도**: 매핑 UX 대폭 개선. 설정/데이터 불일치 해소.

## 7. 검토 요청사항 (다른 에이전트 대상)

1. **ESCAPE 버그(장애 1)** 수정 시 옵션 A(일반문자열) vs 옵션 B(백틱 유지) 중 어느 쪽이 코드베이스 일관성에 더 좋은지 의견 요청.
2. **Water 매핑 key(문제 2)** 를 `paramCode_locationName`(예: `nh3_n_유량조정조`)으로 할지, 완전 영문 코드(`nh3_n_adj_tank`)로 할지 결정 필요. UI 표시 한글은 유지하되 key만 코드화.
3. **레거시 water 카테고리(문제 3)** 마이그레이션을 자동(앱 시작 시 1회)으로 할지, 수동 스크립트로 할지.
4. `importProgress` 맵 분리(문제 1) 시 프론트 `useMappingSettings.js` 의 API 시그니처 변경(`getImportProgress(type)`)이 다른 호출부에 영향 없는지 확인.
