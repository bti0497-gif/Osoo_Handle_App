# 설치버전만들기 (현장 배포용)

이 문서는 `0.9.0` 정식 배포를 기준으로, **최종 수정 완료 후** 설치버전 생성과 GitHub 릴리즈 배포를 진행하기 위한 실행 가이드입니다.  
목표는 다음과 같습니다.

- 현장에 전달할 **설치 화면이 있는 정식 설치파일** 제공
- 설치 화면/아이콘에 **회사 로고 반영**
- 설치 후 앱이 **GitHub Releases 기반 자동업데이트**를 받도록 구성

---

## 1) 배포 방식 결정 (권장 고정안)

- 설치 엔진: `electron-builder` + `NSIS`
- 초기 설치: 현장 PC에서 설치파일(`.exe`) 1회 실행
- 업데이트: 앱 내부 `electron-updater`가 GitHub Release에서 자동 수신

> 참고: Inno Setup으로 별도 전환하지 않아도, 현재 NSIS 설정으로 설치 화면/아이콘 브랜딩이 충분히 가능함.

---

## 2) 사전 조건 체크

아래 항목이 모두 충족된 뒤에만 빌드/배포를 시작한다.

- [ ] 기능 수정 최종 완료 (추가 코드 수정 중단 상태)
- [ ] 로컬 앱 실행 정상 확인 (로그인, 주요 메뉴, 데이터 저장/조회)
- [ ] 회원/현장 관리 등 최근 변경 기능 재검증 완료
- [ ] 자동업데이트 대상 GitHub 저장소 접근 가능
- [ ] 릴리즈 업로드 권한(토큰/권한) 확인

---

## 3) 설치 화면 브랜딩 준비

현재 설정 기준 파일:

- 앱 아이콘: `public/icon.ico`
- NSIS 아이콘:
  - `installerIcon`: `public/icon.ico`
  - `uninstallerIcon`: `public/icon.ico`
  - `installerHeaderIcon`: `public/icon.ico`

브랜딩 체크:

- [ ] 회사 로고가 반영된 `ico` 파일 최종본 준비
- [ ] 해상도별 아이콘 품질(작은 크기/큰 크기) 확인
- [ ] 앱 이름(`productName`) 표기 최종 확정
- [ ] 바탕화면/시작메뉴 단축아이콘 표시명 확인

---

## 4) electron-builder 설정 점검

설정 파일: `electron-builder.config.js`

필수 확인 항목:

- [ ] `appId` 확인
- [ ] `productName` 확인 (현장 표기명)
- [ ] `directories.output` = `release`
- [ ] `directories.buildResources` = `build`
- [ ] `win.target`에 `nsis` 포함
- [ ] `nsis.oneClick = false` (설치 화면 표시)
- [ ] `publish.provider = github`
- [ ] `publish.owner`, `publish.repo` 실제 배포 리포지토리와 일치
- [ ] `releaseType = release` 확인

---

## 5) 버전/릴리즈 정책

- [ ] `package.json` 버전을 `0.9.0`으로 확정
- [ ] 태그 정책 확정 (`v0.9.0` 권장)
- [ ] 릴리즈 노트(변경사항/주의사항) 작성
- [ ] 롤백 시 사용할 이전 안정 버전 확인

---

## 6) 빌드 전 최종 검증 시나리오

최소 검증:

- [ ] 앱 시작/로그인/로그아웃 정상
- [ ] 대시보드 위젯 로딩 정상
- [ ] 회원/현장 추가·수정·저장 정상
- [ ] 설정 메뉴(기본/약품/매핑) 동작 정상
- [ ] 앱 재시작 후 주요 상태(필요 항목) 유지 확인

권장 추가 검증:

- [ ] 인터넷 차단 시 앱 기본 동작 영향 확인
- [ ] 재연결 후 동기화/조회 복구 확인
- [ ] 관리자/현장관리자 권한별 화면 차이 확인

---

## 7) 설치파일 빌드

프로젝트 루트에서 실행:

```bash
npm run build
npm run dist
```

산출물 점검(`release` 폴더):

- [ ] 설치파일 `.exe`
- [ ] 자동업데이트 메타파일(`latest.yml` 계열)
- [ ] 블록맵 파일(`.blockmap`)
- [ ] 파일명/버전 표기 정상 (`0.9.0`)

---

## 8) GitHub Release 업로드

- [ ] 새 릴리즈 생성 (`v0.9.0`)
- [ ] 산출물 업로드:
  - [ ] 설치 `.exe`
  - [ ] `latest.yml`(또는 플랫폼별 최신 메타파일)
  - [ ] `.blockmap`
- [ ] 릴리즈 공개(Publish) 처리
- [ ] 업로드 누락 파일 없는지 재확인

---

## 9) 현장 설치 검증 (실기기)

현장 PC 또는 동일 조건 테스트 PC에서:

- [ ] 설치 마법사 화면에 로고/아이콘 정상 표시
- [ ] 설치 경로 선택/설치 완료 정상
- [ ] 바탕화면/시작메뉴 바로가기 생성 확인
- [ ] 앱 최초 실행 정상

자동업데이트 검증:

- [ ] 테스트용 상위 버전 릴리즈 준비(예: `0.9.1-test`)
- [ ] 앱에서 업데이트 감지/다운로드/적용 흐름 확인
- [ ] 실패 시 기존 버전 유지 여부 확인
