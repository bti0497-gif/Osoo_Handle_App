# 배포 전 준비 및 교차점검 문서 (2026-04-13)

## 1) 문서 목적
- 이번 릴리즈에서 반영된 정책/수정사항을 한 번에 공유
- 다른 에이전트(또는 리뷰어)가 빠르게 재검증할 수 있도록 체크포인트 제공

## 2) 이번 릴리즈 운영 정책(확정)
- 회원/현장 마스터 데이터는 중앙(구글시트)에서 직접 관리
- 현장 앱은 로컬 DB 중심 운영
- 이번 버전은 한 앱 = 한 현장 설정 세트 기준으로 운영
- 운영 입력 데이터(유량/수질/약품/키트/시설)만 BigQuery로 주기 동기화
- 성적서 메뉴는 이번 배포에서 UI 노출 비활성(기능 위치/코드는 유지)
- 접속자정보창의 현장 선택 UI는 유지하되, 실제 현장 전환 로직은 연결하지 않음

## 2-1) 이번 버전 제한사항
- 양방향/다현장 사용자가 로그인하더라도, 이번 버전에서는 현장별 설정 세트를 따로 전환해 저장하는 구조를 지원하지 않음
- 접속자정보창의 현장 드롭다운은 차기 버전 호환을 위해 UI만 유지하며, 이번 배포에서는 실제 동작하지 않는 것이 정상임
- 운영상 이번 배포는 현장별로 앱 1세트, 설정 1세트 기준으로 사용해야 함

## 3) 반영 완료 항목 요약

### A. 수질 엑셀 재임포트 및 다회측정 보존
- 파일: server/routes/settingsRoutes.cjs
- 변경 요지:
  - 수질 임포트 시 measurement_group/measurement_order 생성
  - 같은 날짜 내 다회 측정 분리 저장 (excel:YYYY-MM-DD:001 ...)
  - 재임포트 시 해당 날짜의 기존 excel 데이터만 정리 후 재삽입
- 기대 효과:
  - 같은 날짜의 동일 위치 데이터가 덮어써져 유실되는 문제 방지
  - 새 날짜는 추가, 동일 날짜는 의도된 범위 내 갱신

### B. 기본설정 > 일지설정 UX 정책
- 파일: src/features/settings/useSettingsViewModel.js, src/features/settings/SettingsView.jsx
- 변경 요지:
  - 1계열: 내부/외부 반송 유량옵션은 1계열값 매핑 기본 선택
  - 2계열: 옵션 목록(1계열/2계열/1+2합계) 표시 + 초기값은 빈 선택
  - 슬러지반출 기본설정(업체명/기본반출량) 초기값 빈칸 UX 보정

### C. 웹/앱설정 기본값 시드 정리
- 파일: server/database.cjs
- 변경 요지:
  - 도로공사 URL/수질분석앱 URL 기본값 시드 보장
  - QnTECH 사진 루트 기본 경로 보장
  - 계정 ID/비밀번호는 빈값 유지
  - gemini_api 자격정보 레코드 기본 생성(빈값)

### D. 소통게시판 일시 NaN 표시 수정
- 파일: src/features/board/BoardView.jsx, src/features/board/useBoardViewModel.js
- 변경 요지:
  - BigQuery timestamp 객체형(value/timestampValue/seconds+nanos) 파싱 보강
  - 목록 표시/상세 표시/정렬 모두 안전 파싱 적용
- 기대 효과:
  - 일시가 NaN-NaN으로 깨지는 현상 제거

### E. 성적서 메뉴 비노출(이번 배포 한정)
- 파일: src/components/Sidebar.jsx, src/core/constants/index.js
- 변경 요지:
  - 메뉴 데이터는 유지
  - 사이드바 렌더링에서 certificate 항목만 필터링해 숨김
- 기대 효과:
  - 향후 자동업데이트 시 필터 제거만으로 재활성 가능

### F. 회원/현장 외부 동기화 기본 비활성
- 파일: server/routes/settingsRoutes.cjs
- 변경 요지:
  - 아래 환경변수가 true일 때만 동기화 허용
    - ENABLE_SITE_SYNC_TO_SHEETS
    - ENABLE_INITIAL_SYNC_TO_SHEETS
    - ENABLE_SITE_MEMBER_BIGQUERY_SYNC
  - 기본값(false)에서는 회원/현장 외부 동기화 안 함
- 기대 효과:
  - 현장앱에서 불필요한 마스터 동기화 차단

## 4) 현재 동기화 구조(요약)

### 4-1. 운영 데이터 -> BigQuery
- 대상: flow_readings, medicine_logs, water_quality, kit_logs, facility_logs
- 방식:
  - 로컬 저장 시 is_synced=0
  - 스케줄러가 주기(10분)로 BigQuery 업로드
  - 성공 시 is_synced=1, 실패 시 is_synced=0 롤백(다음 주기 재시도)
- 스케줄러 활성 조건:
  - BIGQUERY_SYNC_ENABLED=true

### 4-2. 소통게시판
- posts/comments를 BigQuery 직접 사용 (로컬 fallback 없음)

### 4-3. 회원/현장(마스터)
- 중앙에서 직접 관리
- 앱 쪽 outbound sync는 기본 비활성

- [ ] 슬러지반출 기본설정(업체명/기본반출량) 초기 진입 시 빈칸 표시 확인

- [ ] 네트워크/권한 오류 상황에서 동기화 실패 시 다음 주기에 재시도되는지 확인
