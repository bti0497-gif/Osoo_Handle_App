# 배포 전 준비 및 교차점검 문서 (2026-04-13)

## 1) 문서 목적
- 이번 릴리즈에서 반영된 정책/수정사항을 한 번에 공유
- 다른 에이전트(또는 리뷰어)가 빠르게 재검증할 수 있도록 체크포인트 제공

## 2) 이번 릴리즈 운영 정책(확정)
- 회원/현장 마스터 데이터는 중앙(구글시트)에서 직접 관리
- 현장 앱은 로컬 DB 중심 운영
- 이번 버전은 한 앱 = 한 현장 설정 세트 기준으로 운영
- 운영 입력 데이터(유량/수질/약품/키트/시설)만 BigQuery로 주기 동기화
- 성적서 메뉴는 이번 배포에서 UI 노출 비활성(기능 위치/코드는 유지)
- 접속자정보창의 현장 선택 UI는 유지하되, 실제 현장 전환 로직은 연결하지 않음

## 2-1) 이번 버전 제한사항
- 양방향/다현장 사용자가 로그인하더라도, 이번 버전에서는 현장별 설정 세트를 따로 전환해 저장하는 구조를 지원하지 않음
- 접속자정보창의 현장 드롭다운은 차기 버전 호환을 위해 UI만 유지하며, 이번 배포에서는 실제 동작하지 않는 것이 정상임
- 운영상 이번 배포는 현장별로 앱 1세트, 설정 1세트 기준으로 사용해야 함

## 4) 현재 동기화 구조(요약)

### 4-1. 운영 데이터 -> BigQuery
- 대상: flow_readings, medicine_logs, water_quality, kit_logs, facility_logs
- 방식:
  - 로컬 저장 시 is_synced=0
  - 스케줄러가 주기(10분)로 BigQuery 업로드
  - 성공 시 is_synced=1, 실패 시 is_synced=0 롤백(다음 주기 재시도)
- 스케줄러 활성 조건:
  - BIGQUERY_SYNC_ENABLED=true

### 4-2. 소통게시판
- posts/comments를 BigQuery 직접 사용 (로컬 fallback 없음)

### 4-3. 회원/현장(마스터)
- 중앙에서 직접 관리
- 앱 쪽 outbound sync는 기본 비활성
