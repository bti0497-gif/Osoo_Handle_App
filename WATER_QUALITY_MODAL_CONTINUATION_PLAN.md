# 수질분석 통합 입력 모달 이어서 작업 계획

작성일: 2026-06-11

## 목표

유량관리, 약품관리, 키트관리 탭의 통합 입력 UX를 정리한 흐름에 맞춰 `수질분석` 탭도 완성한다.

수질분석 탭은 단순 수동 입력만이 아니라 QnTECH 값/사진 불러오기, 로컬 저장, Drive 업로드, 회차 선택/추가, 수동 회차 저장까지 포함하므로 별도 작업 단위로 진행한다.

## 현재 이해한 업무 흐름

1. 수질분석 메인 워크스페이스는 읽기 전용 그리드로 둔다.
2. 수동 추가/수정, QnTECH 불러오기, 회차 추가는 `통합 데이터 입력` 모달의 `수질분석` 탭에서 처리한다.
3. 모달의 데이트피커 날짜가 모든 수질분석 작업의 기준 날짜다.
4. 기존 그리드 행을 더블클릭해서 모달을 열면 해당 날짜/회차 데이터가 이미 있으므로 수정 모드로 바인딩한다.
5. `통합 데이터 작성` 버튼으로 열면 선택된 행에 값이 있으면 수정 모드, 값이 없으면 추가 모드로 연다.
6. `회차 추가` 버튼을 누르면 현재 날짜의 다음 회차가 콤보박스에 자동 추가되고 바로 선택된다.
7. 수동 회차 추가 시 입력 텍스트박스는 모두 빈 값으로 초기화한다.
8. QnTECH 버튼은 단순 값 조회가 아니라 다음 작업을 수행한다.
   - QnTECH 접속
   - 해당 날짜 또는 기간의 분석값 조회
   - 분석 사진 다운로드
   - 로컬 QnTECH 사진 저장 경로에 저장
   - Drive 업로드
   - `qntech_water_quality`에 값 저장
   - BigQuery 동기화 트리거
   - 모달 회차 콤보박스 및 현재 날짜 데이터 재바인딩

## QnTECH 버튼 기능 정리

수질분석 탭의 `QnTECH` 버튼은 기존 수질분석 워크스페이스의 `QnTECH 불러오기` 기능을 모달로 옮긴 것이다.

새 API를 만들기보다 기존 흐름을 재사용한다.

관련 기존 프론트 흐름:

```text
src/features/water/WaterQualityView.jsx
handleQntechImportClick()
handleImportFromQntech()
```

관련 기존 Model/API:

```text
src/features/water/WaterQualityModel.js
POST /api/water-quality/import-from-qntech
POST /api/water-quality/import-range-from-qntech
```

관련 기존 서버 흐름:

```text
server/routes/waterQualityRoutes.cjs
server/services/qntechWaterImportService.cjs
server/services/qntechWaterValueImportService.cjs
server/services/qntechWaterPhotoImportService.cjs
```

## 하루 불러오기와 기간 불러오기

현재 모달 안 QnTECH 버튼은 데이트피커의 하루 날짜를 기준으로 동작하는 형태로 정리되어 있다.

하지만 기존 워크스페이스에는 여러 날짜를 한꺼번에 불러오는 기간 불러오기 기능이 있었다. 이 기능을 없애면 안 된다.

내일 구현 방향:

1. 수질분석 탭 안에 기본 QnTECH 버튼은 유지한다.
   - 기준: 데이트피커 날짜 하루
   - 성공 후 해당 날짜의 회차 목록과 값 재바인딩
2. `기간 불러오기` 기능을 모달 안에 유지한다.
   - 시작일/종료일 입력은 작은 팝오버 또는 접힘 영역으로 배치
   - 기존 `handleQntechImportClick()`의 기간 처리 로직 재사용
   - 여러 날짜의 값/사진 저장 및 Drive 업로드 수행
   - 완료 후 현재 데이트피커 날짜의 회차 목록만 모달에 표시

## 회차 콤보박스 UX

기본 규칙:

1. 날짜에 이미 1~3회차 데이터가 있으면 콤보박스에는 `1회차`, `2회차`, `3회차`가 표시된다.
2. 기본 선택은 `1회차`다.
3. QnTECH에서 값을 불러온 직후에는 해당 날짜의 회차 목록을 다시 읽고 `1회차` 또는 방금 불러온 대표 회차를 선택한다.
4. 콤보박스 회차를 바꾸면 해당 회차의 분석장소별 수질값이 텍스트박스에 바인딩된다.
5. `회차 추가`를 누르면 다음 회차가 자동 추가된다.
   - 예: 1~3회차가 있으면 4회차 추가
   - 4회차가 즉시 선택됨
   - 모든 입력 텍스트박스는 빈 값으로 초기화됨

## 수동 회차 저장 규칙

완전히 빈 신규 회차:

1. 회차 추가 후 아무 값도 입력하지 않고 `저장하기`를 누르면 저장하지 않는다.
2. 메시지: `4회차 입력 데이터가 없습니다.`
3. 기존 1~3회차는 그대로 유지한다.

일부만 입력된 신규 회차:

1. 누락 항목이 있으면 확인 메시지를 띄운다.
2. 메시지 예: `유량조정조의 암모니아성질소 값이 없습니다. 그대로 저장하시겠습니까?`
3. 사용자가 `예`를 누르면 빈 값은 0으로 저장한다.
4. 사용자가 `아니요`를 누르면 가장 먼저 누락된 항목의 분석장소로 자동 이동한다.

기존 회차 수정:

1. 기존 값이 바인딩된 상태에서 수정한다.
2. `저장하기`를 누르면 현재 회차의 전체 값을 upsert한다.
3. 수정 모드에서는 자동 누락 차단보다 기존 데이터 유지/업데이트를 우선한다.

## PO4-P / 인산염인 입력 대상 이슈

현재 문제:

MBR 공법으로 설정되어 있고 분석장소 리스트는 다음처럼 정상적으로 구성되어 있다.

```text
유량조정조
무산소조
포기조
방류조
```

하지만 수질분석 모달에서 모든 장소가 `이 분석장소는 인산염인 입력 대상이 아닙니다.`로 표시되는 문제가 있다.

확인한 원인 후보:

```text
src/features/water/WaterQualityView.jsx
po4pLocations 배열의 장소명이 깨진 문자열로 남아 있음
```

현재 코드상 PO4-P 대상 판정은 다음 흐름이다.

```text
po4pApplicable: po4pLocations.includes(loc.name)
```

`loc.name`은 정상 한글인데 `po4pLocations` 값이 깨져 있으면 `includes()`가 전부 실패한다.

내일 첫 수정 항목:

1. `po4pLocations`를 정상 한글로 교체한다.
2. 공법별 PO4-P 대상은 다음 기준으로 맞춘다.

```text
A2O: 유량조정조, 침전조, 방류조
MBR: 유량조정조, 포기조, 방류조
```

3. `무산소조`는 PO4-P 입력 대상이 아니다.
4. MBR에서 `유량조정조`, `포기조`, `방류조`는 PO4-P 입력칸이 보여야 한다.
5. `무산소조`에만 `이 분석장소는 인산염인 입력 대상이 아닙니다.` 안내가 보여야 한다.

## 구현 순서

1. PO4-P 대상 장소 문자열 깨짐 수정
2. 수질분석 모달의 회차 콤보박스 동작 점검
3. QnTECH 하루 불러오기 버튼을 기존 값/사진/Drive 업로드 흐름에 정확히 연결
4. 기간 불러오기 UI를 모달 안에 배치
5. QnTECH 불러오기 성공 후 해당 날짜 회차 목록 재조회/재바인딩
6. 회차 추가 시 다음 수동 회차 생성 및 빈 입력폼 바인딩
7. 빈 회차 저장 방지
8. 일부 누락 수동 회차 저장 확인창 구현
9. 저장 후 `qntech_water_quality` 로컬 저장 및 BigQuery 동기화 확인
10. 키트관리 `분석키트 동기화`와 수질분석 QnTECH 불러오기 흐름 관계 재확인

## 내일 검증 체크리스트

1. MBR에서 PO4-P 입력 가능 장소가 `유량조정조`, `포기조`, `방류조`로 표시되는가
2. MBR에서 `무산소조`만 PO4-P 입력 대상 제외 안내가 표시되는가
3. 같은 날짜의 1~3회차 QnTECH 데이터가 콤보박스에 표시되는가
4. 회차 변경 시 해당 회차 값이 정확히 바인딩되는가
5. 회차 추가 시 4회차가 추가되고 입력칸이 빈 값으로 초기화되는가
6. 빈 4회차는 저장되지 않는가
7. 일부 누락 4회차는 확인창 후 0 저장 또는 누락 항목 이동이 되는가
8. QnTECH 하루 불러오기가 값, 사진, 로컬 저장, Drive 업로드까지 수행하는가
9. QnTECH 기간 불러오기가 기존처럼 여러 날짜를 처리하는가
10. 저장 후 그리드 새로고침/바인딩이 정상인가
