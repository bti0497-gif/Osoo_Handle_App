# 도로공사 입력도우미 사이트 구조 수집 보고서

## 1. 현재 상태

오늘 작업에서 다음 흐름까지 확인했다.

1. `일지 > 공사 입력 도우미` 메뉴에서 도로공사 사이트가 webview로 표시된다.
2. 설정 메뉴의 도로공사 웹/앱 계정 정보가 정본 로컬 DB에서 조회된다.
3. 로그인 화면의 아이디/비밀번호 필드에 계정 정보가 자동 입력된다.
4. 로그인 버튼은 사용자가 직접 누르는 방식으로 유지한다.
5. 개발 디버그 패널에서 계정 상태 조회와 DOM 저장을 수행할 수 있다.

## 2. 정본 로컬 DB

업무 데이터와 설정 데이터의 기준 DB는 다음 하나로 통일한다.

```text
%APPDATA%\Osoo_Handle_App\osoo.db
```

이 DB에 저장되는 대상은 다음과 같다.

```text
유량
전력량
약품
키트
수질/분석치
설정 메뉴 매핑값
웹/앱 계정 정보
도로공사 로그인 정보
```

Electron `app.getPath('userData')`는 브라우저 세션, 캐시, Preferences 등 런타임 저장소로만 취급한다. 업무 데이터 조회에는 사용하지 않는다.

## 3. 개발 디버그 패널

공사 입력 도우미 화면의 개발 디버그 패널에는 다음 버튼이 있다.

```text
새로고침: 현재 도로공사 webview 새로고침
DevTools: 도로공사 webview 개발자도구 열기
DOM 저장: 현재 도로공사 화면 HTML 구조 저장
계정 상태: 로컬 DB에 계정 정보가 있는지만 확인
```

계정 원문은 화면이나 콘솔에 표시하지 않는다.

저장 위치:

```text
%APPDATA%\Osoo_Handle_App\roadwork-debug\
```

저장 파일 형식:

```text
YYYYMMDD-HHmmss-label.html
YYYYMMDD-HHmmss-label.meta.json
```

## 4. 일요일 작업 시 DOM 저장 권장 순서

도로공사 사이트는 로그인 후 운영일지 메뉴에 들어가면 한 화면에 대부분의 입력 항목이 모여 있다. 그래도 다음 상태별로 DOM을 저장하면 자동입력 흐름을 확정하기 쉽다.

1. 로그인 화면
2. 2차 인증 화면
3. 2차 인증 성공 후 메인 또는 목록 화면
4. 운영일지 메뉴 진입 직후 화면
5. `신규` 클릭 직후 화면
6. 날짜 선택 직후 화면
7. 근무자 이름과 근무인원 입력 후 화면
8. 유량, 전력량, 약품, 키트, 분석사진, 체크리스트가 보이는 실제 입력 화면
9. 저장 버튼을 누르기 직전 최종 상태

입력 화면이 크게 변하지 않으면 모든 단계에서 저장할 필요는 없다. 화면 상태가 바뀌거나 입력칸 활성화 상태가 바뀔 때 저장한다.

## 5. 운영일지 입력 화면 업무 규칙

운영일지 입력은 다음 업무 흐름을 기준으로 설계한다.

1. 현장관리자는 앱에서 해당 날짜의 유량, 약품, 키트, 분석치 입력을 먼저 완료한다.
2. 이후 도로공사 입력도우미 메뉴에 들어와 도로공사 사이트 입력을 진행한다.
3. 쉬는 날 또는 미입력일이 있으면 다음 근무일에 여러 날짜를 몰아서 입력할 수 있다.
4. 날짜는 오늘 날짜만 기준으로 삼지 않고, 사용자가 입력 대상 날짜를 선택할 수 있어야 한다.
5. 도로공사 사이트의 기존 입력 목록 날짜와 로컬 DB의 입력 가능 날짜를 비교한다.

자동입력 판단 기준:

```text
사이트에 해당 날짜가 이미 있으면 입력하지 않는다.
사이트에 해당 날짜가 없고 로컬 DB에 데이터가 있으면 입력 가능하다.
사이트에 해당 날짜가 없고 로컬 DB에도 데이터가 없으면 입력하지 않는다.
```

사용자 메시지 기준:

```text
이미 도로공사 사이트에 등록된 날짜입니다.
입력할 데이터가 없습니다. 먼저 앱에서 해당 날짜의 유량/약품/분석 데이터를 입력해 주세요.
```

## 6. 우선 입력 대상

자동입력 우선 대상:

```text
날짜
근무자 이름
근무인원
유량
전력량
약품 사용현황
키트/분석치 관련 항목
분석사진 첨부 영역
체크리스트
특이사항
```

관례적으로 입력하지 않는 항목:

```text
6. 약품관리대장
7. 슬러지처리관리대장
```

위 두 항목은 우선 자동입력 대상에서 제외한다. 단, DOM 구조는 저장해두고 보고서에 "입력하지 않는 영역"으로 명확히 남긴다.

## 7. 다음 구현 계획

1. 도로공사 운영일지 목록에서 기존 등록 날짜를 추출한다.
2. 로컬 DB에서 입력 가능한 날짜와 데이터 존재 여부를 조회한다.
3. 날짜별 상태 목록을 만든다.
4. 입력 가능한 날짜만 자동입력 버튼을 활성화한다.
5. `신규` 클릭, 날짜 선택, 주요 입력칸 채우기, 체크리스트 반영 순서로 자동입력 흐름을 설계한다.
6. 저장 버튼은 초기에는 자동 클릭하지 않고, 사용자가 최종 확인 후 직접 누르게 둔다.
