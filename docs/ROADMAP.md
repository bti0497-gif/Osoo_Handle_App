# 향후 작업 로드맵

## 1. 2026년 7월 일일업무일지 양식 대응

우선순위가 가장 높은 후속 작업입니다.

- 새 양식 버전을 기존 HWPX 양식과 분리
- 기기류 점검, 기기 마스터와 시설물 수리내역의 최소 데이터 구조 설계
- 기존 유량·수질·약품·운전상태 데이터 재사용
- 신규 책갈피와 공법별 바인딩 규칙 정의
- 양식 버전별 출력 서비스와 회귀 테스트 유지

DB 스키마 변경이 필요하면 구현 전에 사용자 승인을 받습니다.

## 2. 시설물 체크리스트 관리 메뉴

현재 일일업무일지는 공법별 기본 `O` 표시와 빈칸 처리를 수행합니다. 향후 별도 메뉴에서 실제 점검 이력과 시설물 이력카드를 관리합니다.

- 공법별 시설 마스터
- 일일 점검 결과와 이상 항목
- 고장·수리·교체 이력
- 일일업무일지 체크리스트 연동

## 3. 도로공사 입력도우미

현재 웹뷰, 로그인 정보 주입, DOM 수집과 수동 입력 보조 기반까지 구현되어 있습니다.

- WebSquare 컴포넌트 상태 기반 필드 매핑 확정
- 신규 작성 화면의 날짜와 그리드 입력 검증
- 입력 후 사용자가 확인하고 저장하는 안전한 흐름 유지
- 자동 저장은 사이트 API와 컴포넌트 동작이 확정된 뒤 선택 기능으로 추가
- 인증 세션과 사이트 구조 변경에 대한 진단 도구 유지

## 4. 수질 통합 입력 모달

- 날짜별 회차 목록과 수동 회차 추가 UX 정리
- QnTECH 하루/기간 불러오기 구분
- 기존 값 수정과 신규 값 저장 규칙 통일
- MBR/A2O 공법별 PO4-P 및 측정 위치 검증

## 5. 오프라인 백업과 복구

- 미동기화 데이터 백업/복구
- 사진과 서명 문서의 pending 상태 보호
- BigQuery·Drive·Firebase별 재시도 상태 표시

## 6. 릴리즈 확인용 상태바 개선

- 상태표시바에 현재 앱 버전 표시
- 자동업데이트 확인/설치 상태와 현재 실행 버전을 함께 확인할 수 있게 정리
- 현장 배포 직후에는 근무자가 화면만 보고 설치 버전을 확인할 수 있도록 유지

## 7. 설치 시 위치 서비스 최초 1회 안내

- 신규 현장 통합 설치 과정에서 Windows 위치 서비스와 위치 권한 상태 확인
- 위치 서비스가 꺼진 경우에만 Windows 위치 설정 화면을 열어 설치자가 직접 활성화
- 위치 서비스 시작 가능 여부와 권한 상태를 설치 진단 로그에 기록
- 설치 완료 마커를 저장하여 일반 앱 실행과 이후 자동업데이트에서는 안내를 반복하지 않음
- 위치를 읽지 못한 출결은 원격 접속으로 단정하지 않고 `위치 확인 불가`로 구분

## 8. 약품·키트 재고 맞추기

현재는 계획만 보관하며 구현하지 않습니다. 일상 입력과 혼동되지 않도록 약품·키트 관리 화면의 별도 버튼과 모달로 제공합니다.

- 기준 날짜의 실제 재고를 입력하고 장부 재고와 차이를 계산
- 기본 방식은 과거 기록을 바꾸지 않는 `재고 조정` 이력 추가
- 필요할 때만 사용자가 지정한 과거 날짜의 사용량을 수정하고 이후 재고를 다시 계산
- 기간을 지정해 차이를 분산할 경우 균등 또는 기존 사용량 비율 방식을 선택하고, 기본적으로 사용 이력이 있는 날짜에만 반영
- 적용 전에 대상 날짜별 기존값·변경값과 최종 예상 재고를 미리보기로 확인
- 음수 사용량·음수 재고 방지와 품목·단위 일치 검증
- 전체 변경을 하나의 DB 트랜잭션으로 처리하고 한 항목이라도 실패하면 전부 롤백
- 조정 작업 ID, 품목, 기준일, 실제 재고, 차이, 배분 방식·기간, 작업자, 사유와 시각을 감사 이력으로 보존
- 과거 동기화 행을 수정했다면 미동기화 상태로 전환하고, 조정 취소·복구 기능 제공

각 작업은 작은 증분으로 진행하고 `npm run validate`와 `npm run build`를 통과한 뒤 배포합니다.
