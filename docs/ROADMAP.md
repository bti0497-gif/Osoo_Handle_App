# 향후 작업 로드맵

## 1. 2026년 7월 일일업무일지 양식 대응

우선순위가 가장 높은 후속 작업입니다.

- 새 양식 버전을 기존 HWPX 양식과 분리
- 기기류 점검, 기기 마스터와 시설물 수리내역의 최소 데이터 구조 설계
- 기존 유량·수질·약품·운전상태 데이터 재사용
- 신규 책갈피와 공법별 바인딩 규칙 정의
- 양식 버전별 출력 서비스와 회귀 테스트 유지

DB 스키마 변경이 필요하면 구현 전에 사용자 승인을 받습니다.

## 2. 시설물 체크리스트 관리 메뉴

현재 일일업무일지는 공법별 기본 `O` 표시와 빈칸 처리를 수행합니다. 향후 별도 메뉴에서 실제 점검 이력과 시설물 이력카드를 관리합니다.

- 공법별 시설 마스터
- 일일 점검 결과와 이상 항목
- 고장·수리·교체 이력
- 일일업무일지 체크리스트 연동

## 3. 도로공사 입력도우미

현재 웹뷰, 로그인 정보 주입, DOM 수집과 수동 입력 보조 기반까지 구현되어 있습니다.

- WebSquare 컴포넌트 상태 기반 필드 매핑 확정
- 신규 작성 화면의 날짜와 그리드 입력 검증
- 입력 후 사용자가 확인하고 저장하는 안전한 흐름 유지
- 자동 저장은 사이트 API와 컴포넌트 동작이 확정된 뒤 선택 기능으로 추가
- 인증 세션과 사이트 구조 변경에 대한 진단 도구 유지

## 4. 수질 통합 입력 모달

- 날짜별 회차 목록과 수동 회차 추가 UX 정리
- QnTECH 하루/기간 불러오기 구분
- 기존 값 수정과 신규 값 저장 규칙 통일
- MBR/A2O 공법별 PO4-P 및 측정 위치 검증

## 5. 오프라인 백업과 복구

- 미동기화 데이터 백업/복구
- 사진과 서명 문서의 pending 상태 보호
- BigQuery·Drive·Firebase별 재시도 상태 표시

## 6. 릴리즈 확인용 상태바 개선

- 상태표시바에 현재 앱 버전 표시
- 자동업데이트 확인/설치 상태와 현재 실행 버전을 함께 확인할 수 있게 정리
- 현장 배포 직후에는 근무자가 화면만 보고 설치 버전을 확인할 수 있도록 유지

각 작업은 작은 증분으로 진행하고 `npm run validate`와 `npm run build`를 통과한 뒤 배포합니다.
