# Osoo Handle App 현장 배포 방법

## 1. 현장에 가져갈 파일

현장에는 다음 통합 설치파일 **하나만** 가져갑니다.

```text
release\integrated-deployment\
└─ Osoo Handle App Integrated Setup 1.0.3.exe
```

이 파일에는 다음 항목이 모두 포함되어 있습니다.

- Osoo Handle App 1.0.2 본체 설치파일
- `.env.local`
- Google 서비스 계정
- BigQuery 서비스 계정
- Firebase 서비스 계정
- Google OAuth 클라이언트 설정

통합 설치파일은 공통 서비스 자격증명을 포함하므로 GitHub, 메일, 공개 클라우드 또는 메신저에 올리지 않습니다. 직접 관리하는 USB 등으로 현장에 가져갑니다.

## 2. 설치 실행

1. 실행 중인 Osoo Handle App이 있다면 종료합니다.
2. `Osoo Handle App Integrated Setup 1.0.3.exe`를 실행합니다.
3. 설치 위치는 현재 Windows 사용자 영역으로 자동 지정되며 별도로 선택하지 않습니다.
4. 설치 완료 화면이 표시되면 종료합니다.

통합 설치파일은 자동으로 공통 서비스 설정을 다음 위치에 등록합니다.

```text
%APPDATA%\wastewater-treatment-plant\config
```

앱 본체와 자격증명은 하나의 설치 과정에서 함께 설치됩니다. 별도의 두 번째 설치창은 열리지 않습니다.

## 3. 설치 후 현장 지정

1. Osoo Handle App을 실행합니다.
2. Google Sheets에 등록된 admin 계정으로 로그인합니다.
3. admin에게만 표시되는 설정 메뉴를 엽니다.
4. 해당 PC에서 사용할 현장명과 현장관리자를 지정합니다.
5. 설정을 저장한 뒤 로그아웃합니다.
6. 지정한 현장관리자 계정으로 다시 로그인해 현장 정보가 맞는지 확인합니다.

통합 설치파일은 모든 현장에서 공통으로 사용합니다. 현장 구분은 설치파일이 아니라 설치 후 admin 설정에서 결정합니다.

## 4. 설치 후 확인

다음 항목을 확인합니다.

1. 로그인 화면이 정상적으로 열리는지 확인
2. 설정한 현장명과 현장관리자 정보 확인
3. 유량·약품·키트·수질 메뉴가 정상적으로 열리는지 확인
4. 통합 데이터 입력창이 정상적으로 열리는지 확인
5. 서버 연결 실패 표시가 없는지 확인

배포 전 서버 데이터를 정리했으므로 실제 운영 데이터를 시험 입력할 필요는 없습니다. 화면, 현장 설정, 메뉴 접근만 확인합니다.

## 5. 설치파일 보호

설치 완료 후 USB에 담긴 통합 설치파일은 안전하게 보관합니다. 분실 우려가 있으면 USB에서 삭제하고 관리자 보관 장소의 원본만 유지합니다.

설치된 자격증명은 다음 위치에 있으므로 통합 설치파일을 삭제해도 앱 실행에는 영향이 없습니다.

```text
%APPDATA%\wastewater-treatment-plant\config
```

PC 재설치나 장애 복구를 위해 통합 설치파일 원본은 별도의 안전한 저장장치에 보관합니다.

## 6. 자동 업데이트

- 최초 설치 버전은 `1.0.2`입니다.
- 최초 설치 후 앱 업데이트는 GitHub Release에 올라간 앱 본체 버전으로 진행됩니다.
- 통합 설치파일을 다시 만들거나 다시 실행할 필요가 없습니다.
- 다음 업데이트는 `1.0.3` 이상으로 배포합니다.
- AppData의 현장 설정, 공통 자격증명, 로컬 데이터는 앱 업데이트 후에도 유지됩니다.
- 인터넷이 끊겨도 업무 데이터는 로컬에 먼저 저장됩니다.
- BigQuery 전송은 마지막 저장 후 30분 동안 추가 입력이 없을 때 실행됩니다.
- 전송하지 못한 데이터는 다음 유휴 시간이나 다음 앱 실행 때 다시 전송됩니다.

## 6-1. 사진 저장과 Drive 확인

- 수질분석, 슬러지반출, 청소필증, 약품입고, 키트입고 사진은 로컬에 먼저 저장됩니다.
- Drive에는 `관리사진\연도\월` 폴더로 업로드됩니다.
- 파일명은 `날짜_현장명_업무항목.jpg` 형식을 사용하며 여러 장이면 항목 뒤에 순번이 붙습니다.
- 성적서는 중앙관리자 앱이 관리하는 기존 `성적서\연도\월` 구조를 유지합니다.
- Drive 업로드가 실패해도 로컬 저장 데이터는 유지되며 앱에 경고가 표시됩니다.

## 7. 설치 실패 시 확인

### Windows 보안 경고가 표시되는 경우

파일명이 `Osoo Handle App Integrated Setup 1.0.3.exe`인지 확인합니다. 관리자가 직접 만든 배포용 파일이 맞다면 추가 정보 또는 실행 허용을 선택합니다.

### 설치가 취소되었다고 표시되는 경우

기존의 이중 설치창 방식은 사용하지 않습니다. 파일의 수정 날짜가 2026년 6월 23일 이후인지 확인하고, 새 통합 설치파일을 사용합니다.

### 앱이 서버에 연결되지 않는 경우

1. 앱을 완전히 종료합니다.
2. 다시 실행합니다.
3. 계속 실패하면 PC를 재부팅합니다.
4. 그래도 해결되지 않으면 오류 화면과 발생 시각을 기록합니다.

### 현장 정보가 잘못 표시되는 경우

admin 계정으로 로그인해 설정 메뉴에서 현장명과 현장관리자를 다시 지정합니다. 통합 설치파일을 다시 만들 필요는 없습니다.

## 8. 비상용 기존 설치 방식

`release\deployment-package` 폴더는 통합 설치파일이 실행되지 않을 때 사용하는 비상용 패키지입니다. 평상시 현장 배포에는 사용하지 않습니다.

비상용 설치 시에는 폴더 전체와 별도 자격증명 파일이 필요하므로 개발 담당자만 사용합니다.
