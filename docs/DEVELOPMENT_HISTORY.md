- 템플릿 기반 Excel 생성과 사진 업로드 보안 처리
- 소통게시판, 근태, 대시보드와 오프라인 우선 처리 도입

- 수질·슬러지반출·청소필증·약품입고·키트입고 사진의 Drive 경로를 `관리사진/년/월`로 통합하고 현장명 기반 파일명 규칙 적용
- 사진은 로컬 저장을 우선 완료하고 Drive 실패는 업무 저장과 분리하여 사용자에게 경고하도록 보강
- 소스, 빌드, ASAR와 실제 Firebase 반환값 검증
- `1.0.2` 설치파일과 자동 업데이트 파일을 GitHub Release로 배포
- GitHub CLI 2.95.0 설치 및 인증 완료

## 주요 아키텍처 결정

### 로컬 우선

업무 입력은 SQLite에 먼저 저장합니다. 인터넷이나 외부 서비스가 중단되어도 현장 업무는 계속할 수 있어야 하며, BigQuery·Drive·Firebase 동기화는 후속 처리합니다.

### 현장별 차이는 설정으로

공법, 측정 장소, 출력 양식, 셀 매핑과 기본값은 가능한 한 코드 분기보다 설정과 템플릿으로 관리합니다.

### MVVM과 서버 서비스 분리

- Model: API 호출
- ViewModel: 상태와 업무 규칙
- View: UI 렌더링
- Route: 요청/응답 조정
- Service: DB, 외부 API, 파일 및 업무 처리

### 출력물 우선 진단 순서

1. `templates/reports/`의 실제 양식
2. 책갈피·셀 매핑
3. 바인딩 데이터
4. 변환 및 PDF 병합

### 자격증명 분리

서비스 계정, OAuth 정보와 `.env.local`은 Git과 설치 ASAR에 포함하지 않습니다. 현장 PC의 AppData 설정 폴더에 별도로 프로비저닝합니다.
